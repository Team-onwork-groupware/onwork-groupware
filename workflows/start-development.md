# Workflow: Start Development from Empty Repository

## Trigger

Use this when a repository is empty or has only README/init files.

## Steps

1. Read the project goal from the user, issue, or meeting note.
2. Read `AGENTS.md`, `plan.md`, and `memory-bank/activeContext.md`.
3. Confirm stack, test command, build command, and CI requirement.
4. Update `plan.md` with first task scope and verification criteria.
5. Create or adjust baseline project files.
6. Run the verification command.
7. Update `memory-bank/progress.md`.
8. Summarize remaining unknowns.

## Required Human Checkpoint

Before implementing actual product code, confirm:

- Tech stack
- Build/test commands
- Repo ownership and branch policy
- Minimum CI requirement
