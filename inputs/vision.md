# Vision — OnWork 0529 보완

## One-Line Summary

OnWork는 인사, 근태, 휴가, 결재, 알림을 하나의 흐름으로 묶고, DevAuto 에이전트 파이프라인으로 명세-구현-검증의 추적성을 증명하는 그룹웨어 프로젝트다.

## Target User

- 직원: 출퇴근, 시간외 근로 신청, 휴가 신청, 알림 확인을 반복적으로 수행한다.
- 팀장/경영진: 결재함에서 휴가, 근태 이상, 시간외 근로, 인사 변경 요청을 상세 확인하고 처리한다.
- 발표/평가자: AI 에이전트가 왜 이런 구조와 기술을 선택했는지, 실제로 검증했는지 확인한다.

## Core Problem

현재 구현은 핵심 골격이 있으나 0529 PDF 명세와 실제 API/UI/검증 문서가 불일치한다. 교수님 피드백도 단순 기능 완성보다 상세 확인 UX, 역할별 화면 정리, 시간대 문제, 성능/품질 검증 근거를 요구한다.

## Success Definition

- `python3 -m devauto validate <onwork>`가 통과한다.
- 0529 API 명세의 canonical endpoint가 백엔드와 프론트 호출에 반영된다.
- 결재함/인사/근태/휴가/온보딩의 주요 사용자 흐름을 테스트와 브라우저 검증으로 설명할 수 있다.
- 핵심 API에 대해 p95/p99, 실패율, 병목 근거를 남긴다.

## Non-Goals

- 연구실 통합 플랫폼 모드의 실제 구현은 v2로 분리한다.
- 외부 API 키, production 배포 secret, 운영 DB 데이터는 다루지 않는다.
- DevAuto 템플릿으로 OnWork 제품 코드를 덮어쓰지 않는다.
