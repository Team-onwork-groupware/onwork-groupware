1. 개요

운영 Base URL
https://onwork-backend.onrender.com/api/v1
로컬 Base URL
http://localhost:8080/api/v1
인증 방식
JWT Bearer 토큰
Authorization: Bearer <access_token>
데이터 포맷
JSON · 필드명 snake_case · UTF-8
시간대 Asia/Seoul (KST)

모든 엔드포인트는 /api/v1 접두어를 가집니다. 본 문서의 경로는 접두어를 생략하고 도메인 단위로 표기합니다. 요청·응답 본문의 JSON 키는 snake_case(예: access_token, start_date)이며, 날짜/시간은 ISO-8601(KST)을 따릅니다.

권한 표기: 공개 인증 불필요 · 인증 로그인 사용자 · MANAGER+ 팀장 이상 · CEO/VP 경영진  |  별칭 동일 기능의 호환 경로

2. 권한 체계 (RBAC)

5단계 역할 계층. 상위 역할은 하위 역할 권한을 포함합니다(RoleHierarchy). 직급(position)과 권한(role)은 별개입니다.

역할(Role)설명대표 권한
CEO대표이사전사 조회, 보상휴가 부여, 운영 배치 실행
VP경영진인사 변경 승인, 보상휴가 부여
HR_MANAGER경영지원팀장인사 변경 요청·임시저장, 입사 등록, 휴가 대행 결재
MANAGER팀장팀 근태/시간외/휴가 결재, 월마감
EMPLOYEE일반 사원본인 근태·휴가 신청, 마이페이지

6. 휴가관리 (Leave) /leave-* · 14

0529 명세 기준 canonical 경로(/leave-balances, /leave-requests, /leave-grants)가 정식이며, /leave/*는 호환용 구경로입니다. 휴가 유형 ID — 1 연차 · 2 보상휴가 · 3 오전반차 · 4 오후반차.

6-1. Canonical (정식)

Method경로설명권한
GET/leave-balances/me?year내 휴가 잔여 일수(유형별).인증
POST/leave-requests휴가 신청. 연차 1일 이하·시작일 3일 이상 여유면 자동 승인, 반차·보상휴가는 결재 필요.인증
GET/leave-requests/me내 휴가 신청 목록.인증
PATCH/leave-requests/{id}/cancel대기 중 휴가 신청 취소.인증
PATCH/leave-requests/{id}/cancel-approved승인된 휴가 취소 시도(정책상 차단).인증
PATCH/leave-requests/{id}/process휴가 승인/보류. 팀장 부재 시 경영지원팀장→경영진으로 자동 대행.결재자
POST/leave-grants보상휴가 부여 — 여러 직원/팀 단위 동시 부여, 부분 성공 지원.CEO/VP
GET/leave-requests/summary휴가 신청/승인/보류 요약 지표.인증

13. 공통 에러 응답

모든 오류는 { "code": ..., "message": ... } 형식으로 반환됩니다.

{
  "code": "LEAVE_INSUFFICIENT_BALANCE",
  "message": "잔여 휴가일수가 부족합니다"
}
HTTPcode의미
401UNAUTHORIZED인증이 필요합니다
401INVALID_CREDENTIALS이메일 또는 비밀번호가 올바르지 않습니다
401ACCOUNT_LOCKED로그인 5회 실패로 30분간 잠금
401INVALID_TOKEN유효하지 않은 토큰
403FORBIDDEN권한이 없습니다
404NOT_FOUND대상을 찾을 수 없습니다
400INVALID_PAYLOAD필수 정보가 누락되었습니다
400DUPLICATE_EMAIL이미 등록된 이메일
400ALREADY_PROCESSED이미 처리된 요청
400CANNOT_SELF_APPROVE본인 요청은 직접 승인 불가
400LEAVE_INSUFFICIENT_BALANCE잔여 휴가일수 부족
400LEAVE_OVERLAP신청 기간 중복
400LEAVE_INVALID_PERIOD휴가 기간이 올바르지 않음(예: 반차는 하루만)
500INTERNAL_ERROR처리 중 오류 발생

14. 주요 요청 본문 스키마

POST /auth/login

{ "email": "daehan@onwork.kr", "password": "********" }
→ 200 { "access_token", "refresh_token", "user": { "id","name","email","role","position","department_name" } }

POST /leave-requests · POST /leave/requests

{
  "leave_type_id": 1,            // 1 연차 · 2 보상휴가 · 3 오전반차 · 4 오후반차
  "start_date": "2026-06-10",
  "end_date": "2026-06-10",      // 반차는 시작일과 동일(당일)
  "reason": "개인 사유"
}

POST /leave-grants (보상휴가 다중·팀 단위 부여)

{
  "user_ids": [12, 15, 18],      // 여러 직원 동시 부여
  "days": 1,                      // 0.5 이상
  "year": 2026,                   // 생략 시 올해
  "reason": "프로젝트 마감 보상"
}
→ 200 { "total", "success_count", "failure_count", "results":[{ "user_id","ok","error" }] }
