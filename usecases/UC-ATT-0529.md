# UC-ATT-0529: 근태 출퇴근/이상/시간외/월마감 정합화

## Goal

근태 API와 UI를 0529 명세에 맞춘다. 서버 시간은 `Asia/Seoul` 기준으로 계산하고 테스트 가능한 `Clock` 주입 구조를 사용한다.

## Actors

- 직원: 출근, 퇴근, 본인 근태 조회, 시간외 신청을 수행한다.
- 관리자/HR: 근태 이상을 확정하고 월별 요약을 확인한다.
- 시스템: 출퇴근 시각, 이상 유형, 승인된 시간외 근무를 계산한다.

## Main Flow

1. 직원이 출근 버튼을 눌러 출근 기록을 생성한다.
2. 직원이 퇴근 버튼을 눌러 `PATCH /api/v1/attendance/clock-out`을 호출한다.
3. 직원은 `GET /api/v1/attendance/me`로 오늘 또는 기간별 본인 근태를 확인한다.
4. 직원은 `/api/v1/attendance/overtime-requests`로 시간외 근무를 신청한다.
5. 관리자는 근태 이상 항목을 확인하고 유형 변경, 미승인 시간외 인정 여부와 함께 확정한다.
6. HR은 `/api/v1/attendance/monthly-summaries`로 월마감 근태 요약을 확인한다.

## Rules

- 날짜와 시간 계산은 `Asia/Seoul` 기준이다.
- 퇴근은 `PATCH` 메서드가 canonical이다. 기존 `POST`가 남더라도 호환 alias로만 취급한다.
- 이상 확정 요청은 확정 유형과 미승인 시간외 인정 여부를 명시할 수 있어야 한다.
- 응답 필드는 `snake_case`를 사용한다.

## Acceptance Criteria

- 출근 후에는 퇴근 버튼 상태와 위치가 명확하다.
- 서버 지역 설정과 무관하게 한국 날짜 기준으로 출퇴근 일자가 계산된다.
- 미승인 시간외 인정 여부가 월별 근태 요약에 반영된다.
- 월별 요약, 시간외 신청, 이상 확정 API 계약 테스트가 통과한다.
