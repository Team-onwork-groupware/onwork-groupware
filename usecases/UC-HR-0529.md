# UC-HR-0529: HR 변경 요청/일괄 승인 명세 정합화

## Goal

0529 명세 기준으로 인사 변경 요청 흐름을 정리한다. 임시저장 `DRAFT` 흐름은 제거하고, CEO/HR 승인자가 최대 50건까지 일괄 승인할 수 있는 `batch-process` API를 구현한다.

## Actors

- 직원: 본인 정보 변경 요청을 생성한다.
- HR/CEO: 변경 요청 목록을 검토하고 승인한다.
- 시스템: 승인 결과, 부분 성공, 감사 추적 정보를 기록한다.

## Main Flow

1. 직원이 인사 변경 요청을 제출한다.
2. 시스템은 요청 상태를 `PENDING`으로 저장한다.
3. 승인자는 변경 요청 목록에서 상세 내용을 확인한다.
4. 승인자가 `POST /api/v1/hr/change-requests/batch-process`로 최대 50건을 `APPROVE` 처리한다.
5. 시스템은 각 건의 성공/실패를 분리해 처리하고 공통 `batch_id`를 기록한다.
6. 프론트는 성공/실패 건수를 사용자에게 표시한다.

## Rules

- 일괄 처리는 `APPROVE`만 허용한다.
- 요청 개수는 1건 이상 50건 이하여야 한다.
- 일부 건이 실패해도 전체 트랜잭션을 무조건 롤백하지 않는다.
- 이미 처리된 요청은 실패 항목으로 반환한다.
- 응답 필드는 `snake_case`를 사용한다.

## Acceptance Criteria

- `DRAFT` 생성/수정/조회 흐름이 신규 명세 경로에서 노출되지 않는다.
- `batch_id`, `success_count`, `failure_count`, `results[]`가 응답된다.
- 승인된 요청은 직원 정보에 반영되고 감사 로그 또는 변경 이력으로 추적 가능하다.
- 직원 권한 UI에서는 인사 관리 탭 대신 본인 중심 마이페이지가 우선 노출된다.
